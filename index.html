About

I am a mid-level software engineer with experience doing application development and machine learning. Originally, I intended to be a mechanical design engineer, but I quickly found out that confining my skills to computer aided design was not going to let me achieve my goal of building whatever I wanted to. With that realization in mind, I switched disciplines to software engineering in 2021 and picked up the skills I needed to succeed.

Since then, I've built out a suite of applications from the ground up to enable engineers and subject matter experts to create and version control engineering drawing notes and engineering drawing note lists. This application suite sees use from over 600 engineers on a daily basis, peaking at 1,100 concurrent users. While this application is sufficiently complex at nearly 300,000 lines of code, I want to expand the reach of the value I can provide to millions via web applications developed independently.

In that pursuit, I obtained a degree in Machine Learning and Artificial Intelligence before branching into the world of web application development. Some of my current (not yet deployed) projects can be found on the 'projects' section of this site.

Your interest is greatly appreciated, and I look forward to speaking with you more if my skills or passions are in-line with what you're looking for!

Work

Installed Applications

Composer and Scribe are the two primary installed applications my team and I have developed. Composer, the note authoring application, exposes an interface for subject matter experts (SME) to create and manage engineering notes. Scribe, the note list authoring application, exposes an interface for design engineers to create lists of notes using a combination of SME authored notes and user-created notes. When compared to the previous note list authoring process, the Composer + Scribe workflow has been measured to save over $4M in it's two year lifespan (as of late 2024).

Web Applications

Composer supports inspection of all SME created notes via a web interface in which simple actions can be performed.

Software Plugins

Along with installed and web applications, I've built plugins for SolidWorks, DraftSight, AutoCAD, TRIOS, Zemax, and Code V. These plugins use various technologies including C#, Visual Basic, VBA, Java, Bash scripts and intra-machine messengers such as ZeroMQ.

REST APIs

I co-developed the Restful API used for the Composer web application with one of my team members. The REST API now sees use by other internal programs that want to read the engineering note database.

Off-The-Shelf App Config and Deployment

I was the primary technical representative for the configuration and deployment of an off the shelf material test data sheet (MTDS) manager. My duties were to make configuration updates to the MTDS's backend to support new workflows, advise on the technical constraints of the MTDS, communicate engineering requests with the MTDS's developers and assist with implementation, and deploy the MTDS to multiple environments using the Apache Tomcat java app server.

Engineering Automation

I've built a handful of small scale installed applications to manipulate PDF documents, transfer and transform data from various applications to various other applications and databases, and automate some CAD tasks that I found to be tedious during my tenure as a mechanical engineer from 2018 to 2021.
